# ChatGPT Tool Call Limiter

一个 Manifest V3 浏览器扩展，用于在 ChatGPT 长会话里隐藏较旧的工具调用，只显示最新 N 个，默认 N=10，从而减少大量工具调用同时渲染造成的页面卡顿。

## 功能

- 默认只显示最新 10 个 `工具调用 n：xxx` / `Tool call n: xxx` 区块。
- 隐藏旧工具调用使用 `display: none`，减少页面布局和绘制压力。
- 支持在扩展弹窗里修改数量，范围 1-100。
- 支持启用 / 禁用。
- 支持页面角标，显示总数、显示数、隐藏数。
- 支持一键“临时显示全部 / 恢复限流”。
- 作用域限定为：
  - `https://chatgpt.com/*`
  - `https://chat.openai.com/*`

## 安装

Chrome / Edge：

1. 下载或克隆本仓库。
2. 打开 `chrome://extensions` 或 `edge://extensions`。
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本仓库目录。
6. 刷新 ChatGPT 页面。

## 使用

安装后打开 ChatGPT 长会话，工具调用列表会自动只保留最新 10 个。页面右下角会显示状态角标，点击角标可以临时显示全部，再点一次恢复限流。

也可以点击浏览器扩展图标，在弹窗里修改“最多显示最新 N 个工具调用”。

## 实现说明

扩展会扫描 ChatGPT 页面中类似以下标题的工具调用区块：

- `工具调用 53：exec_command`
- `Tool call 53: exec_command`

然后按照页面 DOM 顺序保留最后 N 个，给更早的区块添加隐藏样式。ChatGPT 页面 DOM 结构可能变化，所以这里使用了多层启发式选择器，而不是只依赖单个固定 class。

## 限制

这个扩展不能阻止 ChatGPT 前端内部生成工具调用数据，它做的是在 DOM 层隐藏旧工具调用，主要减少布局、绘制和交互层面的卡顿。如果 ChatGPT 官方 DOM 结构大改，可能需要更新选择器。
